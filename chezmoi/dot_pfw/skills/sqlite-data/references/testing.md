# Testing

Prepare your SQLiteData app for testability and write tests to exercise your SQL queries.

## Bootstrap database for tests

### Step 1: Depend on `DependenciesTestSupport` in your test target

Add `swift-dependencies` as an explicit dependency if it isn't there already, and link `DependenciesTestSuport` to test target:

```swift
.testTarget(
  name: "MyTests",
  dependencies: [
    .product(name: "DependenciesTestSupport", package: "swift-dependencies")
  ]
)
```

### Step 2: Use `.dependencies` trait to bootstrap database

```swift
import Dependencies
import DependenciesTestSupport

@Suite(
  .dependencies {
    try $0.bootstrapDatabase()
    ...
  }
)
```

  * **DO** use `try` and not `try!` in test/suite traits because they have a throwing context.
  * Consult `pfw-testing/SKILL.md` skill to learn how to define a base suite that bootstraps the database and from which all other test suites can inherit from.

## Controlling primary UUIDs

For primary-keyed tables that use `DEFAULT uuid()`, you can make things testable.

Consult the `pfw-dependencies/SKILL.md` skill to learn more.

### Step 1: Declare a custom database function in Swift to override it

```swift
import Dependencies

@DatabaseFunction nonisolated func uuid() -> UUID {
  @Dependency(\.uuid) var uuid
  return uuid()
}
```

### Step 2: Install this database function where the database connection is bootstrapped.

Add it to `prepareDatabase`:

```swift
var configuration = Configuration()
configuration.prepareDatabase { db in
  db.add(function: $uuid)
  ...
}
let database = SQLiteData.defaultDatabase(configuration: configuration)
```

### Step 3: Use `.incrementing` UUID generator in tests

```swift
import Dependencies
import DependenciesTestSupport

@Suite(
  .dependency(\.uuid, .incrementing),
  ...
)
struct MyTests {
  ...
}
```

  * **DO NOT** override with a `constant` UUID. Always detect the next autoincrementing UUID instead.

## Seeding data for tests

When seeding data for tests, provide explicit `id`s to construct values using the `UUID` initializer from `Dependencies` that takes an integer, _and_ use negative integers in order to prevent clashes with `UUID`s generated by feature code:

```swift
import Dependencies

@Suite(
  .dependencies {
    $0.uuid = .incrementing
    try $0.bootstrapDatabase()
    try $0.defaultDatabase.write { db in
      try db.seed {
        RemindersList(id: UUID(-1), title: "Personal")
        Reminder(id: UUID(-1), title: "Get milk", remindersListID: UUID(-1), isCompleted: true)
        Reminder(id: UUID(-2), title: "Walk the dog", remindersListID: UUID(-1))

        RemindersList(id: UUID(-2), title: "Business")
        Reminder(id: UUID(-1), title: "Call accountant", remindersListID: UUID(-2))
        Reminder(id: UUID(-2), title: "File taxes", remindersListID: UUID(-2))
      }
    }
  }
)
```

  * Each newly seeded model should have an ID one _less_ than the lowest ID for that type of model.
  * **DO NOT** share seeds or seed helper functions with Xcode previews or app code. Tests must have their own seeds.

## Testing observable models

Use `expectNoDifference` to assert on large, complex state (use `pfw-custom-dump/SKILL.md` skill for more info):

```swift
import Dependencies

@Test
func basics() async throws {
  let model = RemindersModel(remindersListID: UUID(-1))
  expectNoDifference(
    model.reminders,
    [
      Reminder(id: UUID(-1), title: "Get milk", remindersListID: UUID(-1), isCompleted: true),
      Reminder(id: UUID(-2), title: "Walk the dog", remindersListID: UUID(-1))
    ]
  )
}
```

Use `expectDifference` to assert how a large, complex piece of state changes after an action:

```swift
import Dependencies

@Test
func showCompleted() async throws {
  let model = RemindersModel(remindersListID: UUID(-1))

  expectDifference(model.reminders) {
    model.showCompletedButtonTapped()
    try await model.$reminders.load()
  } changes: {
    $0.removeFirst()
  }
  #expect(!model.showCompleted)
}
```

  * Invoke `.load()` on fetched state properties in model before asserting.
  * Refer to `pfw-custom-dump/SKILL.md` skill for more information on how to best use `expectNoDifference` and `expectDifference`.
